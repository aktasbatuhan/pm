GitHub Token

Your agent needs a GitHub Personal Access Token to read issues, PRs, and project boards on your behalf.

How to create one:
  1. Go to github.com/settings/tokens
  2. Click Generate new token (classic)
  3. Select scopes: repo, read:org, project
  4. Click Generate token and copy it

Organization

Select the GitHub organization or username where your project lives. The agent will track issues, PRs, and boards from this org.

Select Project

Choose the GitHub Projects (v2) board your team uses for planning. The agent will use this as its primary source for tasks, sprints, and priorities.

Select Repositories

The agent will analyze selected repos to build a knowledge base — understanding your architecture, code patterns, and team conventions. This helps it give context-aware answers.
